Introduction to Web APIs

Understanding what APIs are and how they enable applications to communicate.

What are Web APIs?

API stands for Application Programming Interface. In simple terms, an API is a set of rules and protocols that allows different software applications to communicate with each other.

Web APIs specifically enable communication between web applications and servers or other web services over the internet. They define how different components should interact, what data formats to use, and what rules to follow.

Why are APIs Important?
  • Data Access - APIs allow your web applications to access data and functionality from external services
  • Integration - They make it possible to integrate third-party services into your applications
  • Modularity - APIs enable a modular approach to software development
  • Efficiency - You can leverage existing services rather than building everything from scratch

Types of Web APIs

Browser APIs

These are built into the web browser and extend its functionality. Examples include:

  • DOM API - For manipulating HTML and CSS
  • Fetch API - For making HTTP requests
  • Geolocation API - For accessing the user's location
  • Web Storage API - For storing data locally
  • Canvas API - For rendering graphics
Third-Party APIs

These are provided by external services and companies. Examples include:

  • Weather APIs - For accessing weather data
  • Social Media APIs - For integrating with platforms like Twitter or Facebook
  • Payment APIs - For processing payments
  • Maps APIs - For embedding maps and location services
  • Database APIs - For accessing cloud databases

RESTful APIs

REST (Representational State Transfer) is an architectural style for designing networked applications. RESTful APIs are the most common type of web API and follow these principles:

  • Client-Server Architecture - Separation of concerns between client and server
  • Stateless - Each request contains all information needed to complete it
  • Cacheable - Responses can be cached to improve performance
  • Uniform Interface - Consistent way to interact with resources
Common HTTP Methods
  • GET - Retrieve data
  • POST - Create new data
  • PUT - Update existing data
  • DELETE - Remove data
API Documentation

When working with APIs, documentation is your best friend. It explains:

  • What endpoints (URLs) are available
  • What data to send and in what format
  • What responses to expect
  • Authentication requirements
  • Rate limits and usage restrictions

JSON Data Format

Working with JavaScript Object Notation, the standard data format for web APIs.

What is JSON?

JSON (JavaScript Object Notation) is a lightweight data-interchange format that's easy for humans to read and write and easy for machines to parse and generate. It's based on a subset of JavaScript syntax but is language-independent and used by virtually all modern programming languages.

JSON has become the standard format for exchanging data between web applications and APIs due to its simplicity, readability, and compatibility with JavaScript.

JSON vs. JavaScript Objects
  • JSON looks similar to JavaScript object literals but has some key differences
  • In JSON, property names must be in double quotes
  • JSON values can only be strings, numbers, objects, arrays, booleans, or null
  • JSON doesn't support functions, undefined, or comments

JSON Syntax

JSON Objects
// JSON object (key names must be in double quotes) { "name": "John Doe", "age": 30, "isStudent": false, "address": { "street": "123 Main St", "city": "New York", "zipCode": "10001" } }
JSON Arrays
// JSON array containing objects [ { "id": 1, "product": "Laptop", "price": 999.99 }, { "id": 2, "product": "Smartphone", "price": 699.99 }, { "id": 3, "product": "Headphones", "price": 149.99 } ]

Working with JSON in JavaScript

JavaScript provides built-in methods for converting between JSON strings and JavaScript objects:

JSON.parse()

Converts a JSON string into a JavaScript object or array.

// JSON string const jsonString = '{"name":"John","age":30,"city":"New York"}';
// Parse JSON string into a JavaScript object const person = JSON.parse(jsonString);
// Now we can work with it as a normal object console.log(person.name); // "John" console.log(person.age); // 30
JSON.stringify()

Converts a JavaScript object or array into a JSON string.

// JavaScript object const person = { name: "Jane", age: 25, hobbies: ["reading", "music", "hiking"] };
// Convert to JSON string const jsonString = JSON.stringify(person);
console.log(jsonString); // '{"name":"Jane","age":25,"hobbies":["reading","music","hiking"]}'

Error Handling with JSON

When parsing JSON, unexpected syntax errors can cause exceptions. Always use try-catch when working with potentially invalid JSON:

try { const data = JSON.parse(jsonString); // Process the data... } catch (error) { console.error("Error parsing JSON:", error); // Handle the error appropriately }

The Fetch API

Making HTTP requests to retrieve data from servers.

What is the Fetch API?

The Fetch API provides a modern interface for making HTTP requests in JavaScript. It's a more powerful and flexible replacement for the older XMLHttpRequest (XHR).

With the Fetch API, you can:

  • Make GET, POST, PUT, DELETE and other HTTP requests
  • Retrieve data from APIs and web servers
  • Send data to servers
  • Handle responses in a structured way
  • Process different content types (JSON, text, blobs, etc.)
Note: The Fetch API returns Promises, making it work well with modern JavaScript practices like async/await for cleaner, more readable code.

Basic Fetch Syntax

The simplest form of a fetch request just needs a URL:

fetch('https://api.example.com/data') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));

Understanding the Fetch Process

Step 1: Make the Request

The fetch() function returns a Promise that resolves to a Response object.

fetch('https://api.example.com/data') .then(response => { // The Response object contains information about the response console.log(response.status); // HTTP status code (e.g., 200, 404) console.log(response.ok); // true if status 200-299 console.log(response.headers); // Headers object // Need to call a method to actually get the data return response.json(); // Parse JSON response });
Step 2: Process the Data

After getting the Response, you need to extract the data using an appropriate method:

  • response.json() - Parse as JSON (most common)
  • response.text() - Get as plain text
  • response.blob() - Get as binary data
  • response.formData() - Get as FormData
  • response.arrayBuffer() - Get as ArrayBuffer

These methods all return Promises that resolve to the extracted data.

.then(data => { // Now you can work with the actual data console.log(data); // Process the data as needed });

Error Handling in Fetch

It's important to understand that the fetch Promise only rejects when a network error occurs. HTTP error status codes (like 404 or 500) do not cause rejections - you need to check the response status manually:

fetch('https://api.example.com/data') .then(response => { if (!response.ok) { throw new Error(`HTTP error! Status: ${response.status}`); } return response.json(); }) .then(data => console.log('Success:', data)) .catch(error => console.error('Error:', error));

Making Different Types of Requests

You can customize fetch requests by passing a configuration object as the second parameter:

// POST request with JSON data fetch('https://api.example.com/users', { method: 'POST', // HTTP method headers: { 'Content-Type': 'application/json', // Specify content type 'Authorization': 'Bearer your-token-here' // Optional: authorization header }, body: JSON.stringify({ name: "John Doe", email: "john@example.com" }) }) .then(response => response.json()) .then(data => console.log('Success:', data)) .catch(error => console.error('Error:', error));

Async/Await

Working with asynchronous JavaScript in a more readable way.

Introduction to Async/Await

Async/await is a modern JavaScript syntax that makes working with Promises much more elegant and readable. It's syntactic sugar on top of Promises, providing a way to write asynchronous code that looks and feels more like synchronous code.

Introduced in ES2017 (ES8), async/await has quickly become the preferred way to handle asynchronous operations in JavaScript due to its clarity and ease of use.

Remember: Async/await is built on top of Promises. Under the hood, it's still using the same Promise-based system, just with a more readable syntax.

The Basics of Async/Await

Async Functions

An async function is declared with the async keyword:

async function getData() { // This function now returns a Promise return "This is some data"; }
// Equivalent to: function getData() { return new Promise(resolve => { resolve("This is some data"); }); }

Any function declared with async automatically returns a Promise, even if you return a non-Promise value.

The Await Keyword

The await keyword can only be used inside an async function. It pauses the execution of the function until the Promise resolves:

async function fetchUserData() { const response = await fetch('https://api.example.com/user'); const userData = await response.json(); return userData; }

The await expression "unwraps" the value from the resolved Promise and assigns it to the variable.

Promises vs Async/Await

Let's compare the traditional Promise syntax with async/await for the same operation:

Using Promises with .then()
function fetchUserData() { return fetch('https://api.example.com/user') .then(response => { if (!response.ok) { throw new Error('Network response was not ok'); } return response.json(); }) .then(userData => { console.log('User data:', userData); return fetch(`https://api.example.com/posts?userId=${userData.id}`); }) .then(response => response.json()) .then(posts => { console.log('User posts:', posts); return { user: userData, posts }; }) .catch(error => { console.error('Error:', error); }); }
Using Async/Await
async function fetchUserData() { try { const response = await fetch('https://api.example.com/user'); if (!response.ok) { throw new Error('Network response was not ok'); } const userData = await response.json(); console.log('User data:', userData); const postsResponse = await fetch(`https://api.example.com/posts?userId=${userData.id}`); const posts = await postsResponse.json(); console.log('User posts:', posts); return { user: userData, posts }; } catch (error) { console.error('Error:', error); } }
Benefits of Async/Await
  • Cleaner Syntax - Code looks more like synchronous code, which is easier to read
  • Error Handling - Can use try/catch blocks instead of .catch() chains
  • Debugging - Easier to set breakpoints and debug when code isn't inside callback functions
  • Sequential Code - Makes it easier to write code that executes in a specific sequence

Error Handling with Async/Await

With async/await, you can use familiar try/catch blocks for error handling:

async function fetchData() { try { const response = await fetch('https://api.example.com/data'); if (!response.ok) { throw new Error(`HTTP error! Status: ${response.status}`); } const data = await response.json(); return data; } catch (error) { console.error('Fetch error:', error); // You could also rethrow the error // throw error; } }

Running Promises in Parallel

While async/await makes sequential code easy, sometimes you want to run operations in parallel for better performance. You can use Promise.all() for this:

async function fetchMultipleResources() { try { // Start all fetch requests at once (in parallel) const userPromise = fetch('https://api.example.com/user').then(res => res.json()); const postsPromise = fetch('https://api.example.com/posts').then(res => res.json()); const commentsPromise = fetch('https://api.example.com/comments').then(res => res.json()); // Wait for all promises to resolve simultaneously const [userData, posts, comments] = await Promise.all([ userPromise, postsPromise, commentsPromise ]); return { userData, posts, comments }; } catch (error) { console.error('Error fetching data:', error); } }
Important Note: await can only be used inside async functions. If you need to use await at the top level of your script, you can either wrap your code in an immediately invoked async function, or use modern JavaScript modules which support top-level await.

Working with API Data

Processing, filtering, and displaying data from external APIs.

The API Data Workflow

When working with data from APIs, you'll typically follow these steps:

  1. Fetch - Retrieve data from the API
  2. Process - Transform, filter, or organize the data
  3. Display - Present the processed data in your application
  4. Update - Refresh the data or respond to user interaction

Each step involves different techniques and considerations to ensure your application handles data efficiently and presents it effectively to users.

Processing API Data

Once you've fetched data from an API, you'll often need to process it before displaying it to the user:

Transforming Data

API responses might not be structured exactly as needed for your UI. Common transformations include:

  • Extracting specific properties
  • Renaming fields for clarity
  • Converting data types (e.g., strings to dates)
  • Calculating derived values
  • Nesting or unnesting data
// Transforming API data const transformedUsers = apiUsers.map(user => ({ id: user.id, name: `${user.firstName} ${user.lastName}`, email: user.email, joinDate: new Date(user.createdAt) }));
Filtering and Sorting

You often need to filter out irrelevant data or sort it in a specific order:

// Filtering for active users only const activeUsers = users.filter(user => user.isActive);
// Sorting products by price (lowest first) const sortedProducts = products.sort((a, b) => a.price - b.price);
// Complex filtering with multiple conditions const relevantItems = items.filter(item => item.category === "electronics" && item.price < 500 && item.inStock );

Working with Nested Data

API responses often contain complex nested structures. Destructuring and dot notation can help access deeply nested data:

// Accessing nested properties const weatherData = { location: { name: "New York", coordinates: { lat: 40.7128, lon: -74.006 } }, current: { temp: 72, conditions: "Partly Cloudy", wind: { speed: 5, direction: "NE" } }, forecast: [ { day: "Monday", high: 75, low: 65 }, { day: "Tuesday", high: 70, low: 60 } ] };
// Dot notation const cityName = weatherData.location.name; const temperature = weatherData.current.temp; const tomorrowHigh = weatherData.forecast[1].high;
// Using destructuring const { location: { name, coordinates }, current } = weatherData; const { temp, conditions, wind: { speed } } = current; console.log(`In ${name}, it's ${temp}°F and ${conditions} with ${speed}mph winds.`);

Handling Arrays of Data

APIs often return arrays of objects that need to be processed or displayed in lists or tables. Array methods like map, filter, reduce, find, and sort are invaluable here:

async function displayUserTable() { try { // Fetch users from an API const response = await fetch('https://api.example.com/users'); const users = await response.json(); // Filter for active users only const activeUsers = users.filter(user => user.active); // Sort by name activeUsers.sort((a, b) => a.name.localeCompare(b.name)); // Create table rows const tableBody = document.getElementById('user-table-body'); tableBody.innerHTML = ''; activeUsers.forEach(user => { const row = document.createElement('tr'); row.innerHTML = ` <td>${user.id}</td> <td>${user.name}</td> <td>${user.email}</td> <td>${new Date(user.joinDate).toLocaleDateString()}</td> `; tableBody.appendChild(row); }); // Update summary information document.getElementById('user-count').textContent = activeUsers.length; } catch (error) { console.error('Error fetching users:', error); document.getElementById('error-message').textContent = 'Failed to load users. Please try again.'; } }

Error Handling and Loading States

When working with API data, always handle possible errors and provide feedback to users during loading:

async function fetchAndDisplayData() { const dataContainer = document.getElementById('data-container'); const loadingIndicator = document.getElementById('loading-indicator'); const errorMessage = document.getElementById('error-message'); // Reset state errorMessage.style.display = 'none'; dataContainer.innerHTML = ''; loadingIndicator.style.display = 'block'; try { const response = await fetch('https://api.example.com/data'); if (!response.ok) { throw new Error(`HTTP error! Status: ${response.status}`); } const data = await response.json(); // Process and display the data data.forEach(item => { const element = document.createElement('div'); element.textContent = item.name; dataContainer.appendChild(element); }); } catch (error) { console.error('Error:', error); errorMessage.textContent = `Failed to load data: ${error.message}`; errorMessage.style.display = 'block'; } finally { // Hide loading indicator when done, regardless of success/failure loadingIndicator.style.display = 'none'; } }

Data-Driven Styling

Dynamically styling elements based on data values.

What is Data-Driven Styling?

Data-driven styling is the practice of dynamically changing the appearance of elements based on the values of your data. This creates a more interactive and informative user experience by helping users quickly understand the data through visual cues.

Common examples include:

  • Color-coding table rows based on status or values
  • Highlighting values that exceed thresholds
  • Adjusting the size of elements proportionally to their values
  • Showing/hiding elements based on data conditions
  • Using icons or symbols to represent different data states

Applying Styles Based on Data

There are several techniques for applying data-driven styles:

Using Inline Styles

You can directly modify the style property of elements:

function stylePriceElements() { const priceElements = document.querySelectorAll('.price'); priceElements.forEach(element => { const price = parseFloat(element.textContent.replace('$', '')); if (price < 10) { element.style.color = 'green'; } else if (price < 50) { element.style.color = 'orange'; } else { element.style.color = 'red'; element.style.fontWeight = 'bold'; } }); }
Using CSS Classes

Often cleaner than inline styles - define classes in CSS and apply them conditionally:

/* CSS */ .price-low { color: green; } .price-medium { color: orange; } .price-high { color: red; font-weight: bold; }
// JavaScript function stylePriceElements() { const priceElements = document.querySelectorAll('.price'); priceElements.forEach(element => { const price = parseFloat(element.textContent.replace('$', '')); // Remove any existing price classes element.classList.remove('price-low', 'price-medium', 'price-high'); // Add appropriate class based on value if (price < 10) { element.classList.add('price-low'); } else if (price < 50) { element.classList.add('price-medium'); } else { element.classList.add('price-high'); } }); }

Conditional Rendering Based on Data

Data can also drive which elements are shown, their content, and their attributes:

function renderProductList(products) { const container = document.getElementById('product-list'); container.innerHTML = ''; products.forEach(product => { const card = document.createElement('div'); card.className = 'product-card'; // Add 'featured' class for featured products if (product.featured) { card.classList.add('featured'); } // Conditionally show a "sale" badge let saleBadge = ''; if (product.onSale) { saleBadge = '<span class="badge bg-danger">SALE</span>'; } // Determine stock status display let stockStatus = ''; if (product.stock === 0) { stockStatus = '<span class="stock-status out">Out of Stock</span>'; } else if (product.stock < 5) { stockStatus = `<span class="stock-status low">Only ${product.stock} left!</span>`; } else { stockStatus = '<span class="stock-status">In Stock</span>'; } // Construct the card content card.innerHTML = ` <div class="product-header"> <h3>${product.name} ${saleBadge}</h3> </div> <div class="product-price" data-price="${product.price}"> $${product.price.toFixed(2)} </div> <div class="product-description"> ${product.description} </div> ${stockStatus} <button class="btn btn-primary" ${product.stock === 0 ? 'disabled' : ''}> ${product.stock === 0 ? 'Sold Out' : 'Add to Cart'} </button> `; container.appendChild(card); }); // Apply price styling after cards are created stylePriceElements(); }

Dynamic Colors and Gradients

For visualizing data, you might need to generate colors programmatically based on values:

Color Scale Functions

Create functions that map numerical values to colors:

// Generate a color between red and green based on value // value should be between 0 (red) and 1 (green) function getColorForValue(value) { // Ensure value is between 0 and 1 value = Math.max(0, Math.min(1, value)); // Calculate RGB components const red = Math.round(255 * (1 - value)); const green = Math.round(255 * value); const blue = 0; return `rgb(${red}, ${green}, ${blue})`; } // Usage example function styleProgressBars() { const bars = document.querySelectorAll('.progress-bar'); bars.forEach(bar => { const value = parseFloat(bar.getAttribute('data-value')); const percent = value / 100; bar.style.width = `${value}%`; bar.style.backgroundColor = getColorForValue(percent); }); }
Generating Gradients

You can also create color gradients for visualization:

// Create a CSS gradient based on a value function createGradient(value, min, max) { // Normalize value between 0 and 1 const normalized = (value - min) / (max - min); const percent = Math.max(0, Math.min(1, normalized)) * 100; // Define gradient colors const lowColor = '#4caf50'; // Green const midColor = '#ffeb3b'; // Yellow const highColor = '#f44336'; // Red // Create a gradient string if (percent <= 50) { // Blend from low to mid const blendPercent = percent * 2; // 0-100% return `linear-gradient(to right, ${lowColor}, ${midColor} ${blendPercent}%, ${highColor})`; } else { // Blend from mid to high const blendPercent = (percent - 50) * 2; // 0-100% return `linear-gradient(to right, ${lowColor}, ${midColor} ${blendPercent}%, ${highColor})`; } }

Visualizing Data with SVG

For more advanced visualizations, you can generate SVG elements programmatically:

function createBarChart(data, container) { const maxValue = Math.max(...data.map(d => d.value)); const width = 500; const height = 300; const barWidth = width / data.length - 10; // Create SVG element const svg = document.createElementNS('http://www.w3.org/2000/svg', 'svg'); svg.setAttribute('width', width); svg.setAttribute('height', height); // Create a bar for each data point data.forEach((item, index) => { const value = item.value; const barHeight = (value / maxValue) * 250; const x = index * (barWidth + 10) + 5; const y = height - barHeight - 20; // Create bar const bar = document.createElementNS('http://www.w3.org/2000/svg', 'rect'); bar.setAttribute('x', x); bar.setAttribute('y', y); bar.setAttribute('width', barWidth); bar.setAttribute('height', barHeight); // Color based on value (red for low, green for high) const colorPercent = value / maxValue; const color = getColorForValue(colorPercent); bar.setAttribute('fill', color); // Add label const text = document.createElementNS('http://www.w3.org/2000/svg', 'text'); text.setAttribute('x', x + barWidth / 2); text.setAttribute('y', height - 5); text.setAttribute('text-anchor', 'middle'); text.setAttribute('font-size', '12px'); text.textContent = item.label; svg.appendChild(bar); svg.appendChild(text); }); container.appendChild(svg); }

Advanced Object Techniques

Destructuring, spread syntax, and more advanced object operations.

Modern JavaScript Object Features

ES6 and later JavaScript versions introduced powerful features for working with objects that can make your code more concise, readable, and flexible. These techniques are especially useful when working with API data, which often involves complex objects and arrays.

Object Destructuring

Destructuring allows you to extract properties from objects into individual variables with minimal code:

Basic Object Destructuring
const user = { name: 'John Doe', email: 'john@example.com', age: 32, address: { city: 'New York', zipCode: '10001' } };
// Traditional property access const userName = user.name; const userEmail = user.email; const userAge = user.age;
// With destructuring (cleaner) const { name, email, age } = user; console.log(name); // 'John Doe' console.log(email); // 'john@example.com' console.log(age); // 32
Nested Destructuring
// Destructuring nested objects const { name, address: { city, zipCode } } = user; console.log(city); // 'New York' console.log(zipCode); // '10001'
// Renaming variables during destructuring const { name: userName, email: userEmail } = user; console.log(userName); // 'John Doe'
// Default values for missing properties const { phone = 'N/A' } = user; console.log(phone); // 'N/A' (since it's not in user)

Rest and Spread Operators

The spread (...) and rest operators provide powerful ways to work with objects and arrays:

Object Spread Operator

The spread operator lets you create shallow copies of objects and merge objects:

const user = { name: 'John', age: 30 };
// Creating a copy of an object const userCopy = { ...user }; console.log(userCopy); // { name: 'John', age: 30 }
// Adding new properties while copying const userWithEmail = { ...user, email: 'john@example.com' }; // { name: 'John', age: 30, email: 'john@example.com' }
// Merging multiple objects const userDetails = { occupation: 'Developer', location: 'New York' }; const completeUser = { ...user, ...userDetails }; /* { name: 'John', age: 30, occupation: 'Developer', location: 'New York' } */
Rest Operator in Objects

The rest operator lets you collect remaining properties into a new object:

const user = { name: 'John', age: 30, email: 'john@example.com', phone: '555-1234', address: '123 Main St' };
// Extract some properties and collect the rest const { name, email, ...otherInfo } = user; console.log(name); // 'John' console.log(email); // 'john@example.com' console.log(otherInfo); // { age: 30, phone: '555-1234', address: '123 Main St' }
// Useful for removing specific properties const { password, ...safeUserData } = userData; // safeUserData has all properties except password

Object Methods

JavaScript provides several useful built-in methods for working with objects:

Method Description Example
Object.keys() Returns an array of a given object's own property names Object.keys(user) → ['name', 'age', 'email']
Object.values() Returns an array of a given object's own property values Object.values(user) → ['John', 30, 'john@example.com']
Object.entries() Returns an array of a given object's own [key, value] pairs Object.entries(user) → [['name', 'John'], ['age', 30], ...]
Object.assign() Copies all enumerable properties from source objects to a target object Object.assign({}, user, { role: 'admin' })
Object.fromEntries() Transforms a list of key-value pairs into an object Object.fromEntries([['name', 'John'], ['age', 30]])

Computed Property Names

You can use expressions in square brackets to create dynamic property names:

const propertyName = 'email'; const timestamp = new Date().getTime();
// Object with computed property names const user = { name: 'John', [propertyName]: 'john@example.com', // 'email': 'john@example.com' [`lastActive_${timestamp}`]: new Date() };

Object Property Shorthand

When variable names match property names, you can use shorthand notation:

const name = 'John'; const age = 30; const email = 'john@example.com';
// Traditional object creation const user1 = { name: name, age: age, email: email };
// Using property shorthand (cleaner) const user2 = { name, age, email }; // Both user1 and user2 have the same properties and values

Method Shorthand

ES6 introduced a shorter way to define methods in objects:

// Traditional method definition const user1 = { name: 'John', greet: function() { return `Hello, my name is ${this.name}`; } };
// Method shorthand const user2 = { name: 'John', greet() { return `Hello, my name is ${this.name}`; } }; // Both do the same thing console.log(user1.greet()); // "Hello, my name is John" console.log(user2.greet()); // "Hello, my name is John"

Optional Chaining

The optional chaining operator (?.) allows you to access deeply nested properties without worrying about whether intermediate properties exist:

const user = { name: 'John', address: { city: 'New York' // No zipCode property } // No employment property };
// Traditional approach (error-prone) let zipCode; if (user && user.address && user.address.zipCode) { zipCode = user.address.zipCode; } else { zipCode = 'Unknown'; }
// Using optional chaining (safer and cleaner) zipCode = user?.address?.zipCode || 'Unknown'; // Accessing deeply nested properties safely const company = user?.employment?.company?.name; // Returns undefined instead of throwing an error

Nullish Coalescing

The nullish coalescing operator (??) provides a way to select the first "defined" value from a list of options:

const user = { name: 'John', age: 30, score: 0, // Note: this is a valid value, not "falsy" displayName: '' // Empty string (falsy but valid) };
// Using OR operator (||) can be problematic with valid falsy values: const userScore = user.score || 100; // 100 (incorrect, because 0 is falsy) const userName = user.displayName || user.name; // "John" (incorrect if empty string is valid)
// Using nullish coalescing (only null/undefined trigger the fallback) const betterScore = user.score ?? 100; // 0 (correct: keeps 0 since it's defined) const betterName = user.displayName ?? user.name; // "" (keeps empty string) const title = user.title ?? "Guest"; // "Guest" (fallback for undefined property)

Real-World Example: Working with API Responses

These advanced object techniques are particularly valuable when working with API data:

async function processUserData() { try { // Fetch user data from an API const response = await fetch('https://api.example.com/users/123'); const userData = await response.json(); // Extract only needed properties with destructuring const { id, name, email, profile: { bio, avatar = 'default.jpg' } = {}, // Default empty object if profile is missing preferences = {} // Default value } = userData; // Remove sensitive data from user object with rest operator const { password, securityQuestions, ...publicUserData } = userData; // Create a new user object with additional defaults using spread const processedUser = { ...publicUserData, displayName: userData.displayName ?? userData.name, isActive: userData.status === 'active', lastLogin: new Date(userData.lastLoginTimestamp ?? Date.now()), settings: { theme: 'light', notifications: true, ...userData?.settings // Merge with any user-defined settings } }; return processedUser; } catch (error) { console.error('Error processing user data:', error); throw error; } }
