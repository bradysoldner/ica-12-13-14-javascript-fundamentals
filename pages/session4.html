Array forEach Method

Using the forEach method to execute a function for each array element.

What is the forEach Method?

The forEach() method executes a provided function once for each array element. It's a cleaner alternative to traditional for loops when you need to perform an operation on each item in an array.

Unlike other array methods like map() or filter(), forEach() always returns undefined and can't be chained with other methods. It's primarily used for its side effects (like modifying the DOM or logging).

const numbers = [1, 2, 3, 4, 5];
// Using forEach to log each number numbers.forEach(function(number) { console.log(number); });
// Same thing with arrow function (more concise) numbers.forEach(number => console.log(number));

For...of Loops

Iterating over arrays and other iterable objects with for...of loops.

What are For...of Loops?

The for...of loop was introduced in ES6 (ECMAScript 2015) and provides a clean, concise way to iterate over iterable objects like arrays, strings, maps, sets, and more.

Unlike the traditional for loop, for...of gives you direct access to the values in the iterable, not just the indexes. This makes your code more readable and less prone to errors.

Basic Syntax
for (let value of iterable) { // code to execute for each value }

Where:

  • value: A variable that receives the current value in each iteration
  • iterable: Any iterable object (array, string, map, set, etc.)
Examples
// Iterating over an array const colors = ["red", "green", "blue"];
for (let color of colors) { console.log(color); }
// Iterating over a string const greeting = "Hello"; for (let char of greeting) { console.log(char); } // "H", "e", "l", "l", "o"

JavaScript Objects

Working with key-value pairs to create structured data.

What are JavaScript Objects?

Objects are one of the most important data types in JavaScript. They allow you to store collections of related data and functionality in a single container. Think of objects as collections of key-value pairs, where each key is a property name and each value can be any valid JavaScript data type (including other objects or functions).

Objects are useful for representing real-world entities with multiple characteristics. For example, a user object might have properties like name, email, age, and preferences.

Creating Objects

There are several ways to create objects in JavaScript:

Object Literals
// Object literal notation (most common) const person = { firstName: "John", lastName: "Doe", age: 30, isStudent: false };
Using the Object Constructor
// Creating an empty object first const car = new Object();
// Adding properties car.make = "Toyota"; car.model = "Corolla"; car.year = 2022;

Accessing Object Properties

You can access object properties in two ways:

// Dot notation console.log(person.firstName); // "John"
// Bracket notation (useful when property names are dynamic or have special characters) console.log(person["lastName"]); // "Doe"
// Using a variable as property name const propertyName = "age"; console.log(person[propertyName]); // 30

Object Methods

Creating and using methods in objects.

What are Object Methods?

Object methods are functions that are stored as object properties. They allow objects to have behavior in addition to data. When a function is part of an object, we call it a method.

Methods can access and modify the object's properties using the this keyword, which refers to the current object instance.

Defining Methods in Objects

Method Syntax in Object Literals
const person = { firstName: "John", lastName: "Doe", age: 30,
// Method that uses object properties getFullName() { return this.firstName + " " + this.lastName; },
// Method with parameters greet(greeting) { return greeting + ", " + this.getFullName() + "!"; } };
Calling Object Methods
// Call the getFullName method console.log(person.getFullName()); // "John Doe"
// Call the greet method with an argument console.log(person.greet("Hello")); // "Hello, John Doe!" console.log(person.greet("Good morning")); // "Good morning, John Doe!"

The 'this' Keyword in Methods Optional

In an object method, this refers to the object that the method belongs to. It allows methods to access and modify the object's properties.

const counter = { count: 0, increment() { this.count++; return this.count; }, decrement() { this.count--; return this.count; }, reset() { this.count = 0; return this.count; } };
counter.increment(); // 1 counter.increment(); // 2 counter.decrement(); // 1 counter.reset(); // 0

Traversing Objects

Using Object.keys() and Object.values() to work with object properties.

Working with Object Properties Dynamically

JavaScript provides several built-in methods that allow you to work with objects in a more dynamic way, accessing keys and values without knowing them in advance.

These methods are particularly useful when you need to:

  • Iterate over all properties in an object
  • Create lists of keys or values
  • Check if an object has certain properties
  • Create new objects based on existing ones

Object.keys(), Object.values(), and Object.entries()

const person = { name: "Jane", age: 28, city: "Boston", isStudent: true };
// Get an array of all keys const keys = Object.keys(person); console.log(keys); // ["name", "age", "city", "isStudent"]
// Get an array of all values const values = Object.values(person); console.log(values); // ["Jane", 28, "Boston", true]
// Get key-value pairs as arrays const entries = Object.entries(person); console.log(entries); /* [ ["name", "Jane"], ["age", 28], ["city", "Boston"], ["isStudent", true] ] */
// Create an object from entries const newPerson = Object.fromEntries(entries); console.log(newPerson); // Same as original person object

Iterating Over Object Properties

You can use Object.keys() with a for...of loop or a .forEach() method to iterate over all properties in an object:

const product = { name: "Laptop", price: 999.99, brand: "TechBrand", inStock:inStock: true, specs: { cpu: "Intel i7", ram: "16GB", storage: "512GB SSD" } };
// Iterate over all properties for (let key of Object.keys(product)) { const value = product[key]; console.log(`${key}: ${value}`); }
// Iterate over all values using forEach Object.values(product).forEach(value => { console.log(value); });

Note: For nested objects (like product.specs above), Object.keys() will only return the top-level keys. To iterate through nested objects, you would need to check if each value is an object and recursively process it.

Working with classList

Dynamically adding, removing, and toggling CSS classes.

The classList API

The classList property provides a convenient way to manipulate an element's CSS classes. Instead of manually parsing and modifying the className string, classList gives you methods to add, remove, toggle, and check for classes.

This makes it much easier to dynamically update an element's appearance based on user interactions or application state.

classList Methods

Method Description Example
add() Adds one or more classes to an element element.classList.add("active", "highlight");
remove() Removes one or more classes from an element element.classList.remove("disabled");
toggle() Adds a class if it doesn't exist, removes it if it does element.classList.toggle("selected");
contains() Checks if an element has a specific class if (element.classList.contains("active")) { ... }
replace() Replaces an existing class with a new class element.classList.replace("old-class", "new-class");

Event Delegation Optional

Managing events for dynamic elements efficiently.

What is Event Delegation?

Event delegation is a technique where you attach a single event listener to a parent element instead of adding event listeners to multiple child elements. The event listener on the parent will fire for all descendants because of event bubbling.

This approach has several benefits:

  • Memory efficiency - Only one event listener needed instead of many
  • Dynamic elements - Works for elements added to the DOM after the initial page load
  • Less code - Cleaner implementation for handling similar elements
  • Automatic handling of added/removed elements - No need to manually add/remove event listeners

How Event Delegation Works

When an event occurs on an element (like a click), the event bubbles up through its ancestors in the DOM tree. Event delegation leverages this bubbling process by placing an event listener on a parent element and using event.target to determine which child element triggered the event.

// Instead of this (adding listeners to each button): const buttons = document.querySelectorAll(".delete-btn"); buttons.forEach(button => { button.addEventListener("click", handleDelete); });
// Use event delegation (single listener on parent): const container = document.getElementById("items-container"); container.addEventListener("click", function(event) { if (event.target.classList.contains("delete-btn")) { // A delete button was clicked handleDelete(event); } });

Event Delegation in Action

Dynamic Item List

// Event delegation implementation document.addEventListener("DOMContentLoaded", function() { const container = document.getElementById("delegation-container");

Form Data Handling

Capturing and processing user input from forms.

Working with HTML Forms

Forms are an essential part of web applications, allowing users to input data that can be processed by JavaScript. Handling form data involves capturing user inputs, validating them, and responding accordingly.

Key Aspects of Form Handling:
  • Preventing the default form submission behavior
  • Accessing form elements and their values
  • Validating user input
  • Displaying error messages
  • Submitting data (to a server or processing it client-side)
  • Providing feedback to the user

Form Event Handling

The most common event to handle for forms is the submit event, which fires when the user attempts to submit the form (by clicking a submit button or pressing Enter in a form field).

const form = document.getElementById("signup-form"); form.addEventListener("submit", function(event) { // Prevent the default form submission event.preventDefault();
// Get form data and process it const username = document.getElementById("username").value; const email = document.getElementById("email").value;
// Validate the data if (username === "" || email === "") { alert("Please fill out all fields"); return; }
// Process the valid data console.log("Form submitted:", username, email); });

The FormData Object

The FormData object provides a convenient way to capture all form inputs without having to select each one individually.

form.addEventListener("submit", function(event) { event.preventDefault();
// Create a FormData object from the form const formData = new FormData(this);
// Access individual form fields const username = formData.get("username"); const email = formData.get("email");
// Convert to a regular object const formValues = Object.fromEntries(formData.entries()); console.log(formValues); });
