Functions Introduction

Functions are reusable blocks of code designed to perform specific tasks.

What are Functions?

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable set of statements that performs a task or calculates a value.

Think of functions as small programs within your program that you can call whenever you need them.

Why Use Functions?
  • Code Reusability - Write once, use many times
  • Organization - Break complex problems into smaller, manageable pieces
  • Maintenance - Easier to update and fix issues
  • Abstraction - Hide complex implementation details

Function Syntax

Here's the basic syntax for declaring a function in JavaScript:

// Function Declaration function functionName() { // Code to be executed // when the function is called } // Function Call (invoking the function) functionName();

Parameters & Arguments

Parameters allow functions to work with different inputs each time they're called.

Parameters are variables listed in the function definition. They act as placeholders for values that will be passed to the function when it's called.

Arguments are the actual values passed to the function when it's called.

Remember: Parameters are defined in the function declaration, while arguments are the actual values passed when the function is called.
// Function with parameters function greet(name, time) { return "Good " + time + ", " + name + "!"; } // Function call with arguments let message = greet("Alice", "morning"); // message = "Good morning, Alice!"

Default Parameters

ES6 introduced default parameters, which allow you to set default values for parameters if no argument is provided or if the argument is undefined.

function greet(name = "Guest", time = "day") { return "Good " + time + ", " + name + "!"; } greet(); // "Good day, Guest!" greet("Bob"); // "Good day, Bob!" greet("Charlie", "evening"); // "Good evening, Charlie!"

Return Values

Functions can send data back to the code that called them using the return statement.

The return statement ends function execution and specifies a value to be returned to the function caller.

If you don't specify a return value, the function returns undefined by default.

Important: Any code after a return statement won't be executed, as the function immediately exits.
function sum(a, b) { return a + b; // Code below will never run console.log("This won't execute"); } let result = sum(5, 3); console.log(result); // 8
function greet() { console.log("Hello!"); // No return statement }
let greeting = greet(); console.log(greeting); // undefined

Function Scope Optional

Scope determines where variables are accessible in your code.

Understanding Scope

In JavaScript, there are two main types of scope:

  • Global Scope - Variables declared outside any function are globally accessible
  • Local Scope - Variables declared inside a function are only accessible within that function

With the introduction of let and const, JavaScript also has block scope (variables declared inside code blocks like loops or conditionals).

// Global variable let globalVar = "I'm global";
function exampleFunction() { // Local variable let localVar = "I'm local"; console.log(globalVar); // Works: "I'm global" console.log(localVar); // Works: "I'm local" }
exampleFunction(); console.log(globalVar); // Works: "I'm global" console.log(localVar); // Error: localVar is not defined

Variable Shadowing

When a local variable has the same name as a global variable, the local variable "shadows" the global one within its scope.

let message = "Global message"; function showMessage() { let message = "Local message"; console.log(message); // "Local message" }
showMessage(); console.log(message); // "Global message"

Arrow Functions Optional

Arrow functions provide a concise syntax for writing functions in JavaScript.

Arrow functions were introduced in ES6 (2015) and offer a shorter syntax compared to traditional function expressions.

They're especially useful for short, simple functions and when you want to preserve the lexical this value.

Note: Arrow functions don't have their own this binding, which makes them unsuitable for methods in objects or constructors.
// Traditional function expression const add = function(a, b) { return a + b; }; // Arrow function equivalent const addArrow = (a, b) => { return a + b; }; // Even shorter for simple returns const addShort = (a, b) => a + b;

Arrow Function Syntax Variations

Case Syntax Example
No parameters () => { ... } const sayHi = () => { console.log("Hi!"); };
One parameter param => { ... }
(parentheses optional)		 const double = num => { return num * 2; };
Multiple parameters (param1, param2) => { ... } const sum = (a, b) => { return a + b; };
Simple return expression param => expression
(implicit return)		 const square = num => num * num;
Returning an object () => ({ ... })
(parentheses required)		 const getPerson = () => ({ name: "John", age: 30 });

DOM Interaction

Selecting and modifying elements in the Document Object Model.

Selecting DOM Elements

JavaScript provides several methods to select elements from the DOM:

  • getElementById() - Select by ID
  • getElementsByClassName() - Select by class name
  • getElementsByTagName() - Select by tag name
  • querySelector() - Select first matching CSS selector
  • querySelectorAll() - Select all matching CSS selectors
Modifying DOM Elements

Once selected, you can modify elements in various ways:

  • element.innerHTML - Change HTML content
  • element.textContent - Change text content
  • element.style.property - Change CSS styles
  • element.setAttribute() - Set attribute values
  • element.classList - Add/remove CSS classes

DOM Selection in Action

// Select by ID const header = document.getElementById("main-header"); // Select by class name const paragraphs = document.getElementsByClassName("content-paragraph"); // Select by tag name const buttons = document.getElementsByTagName("button"); // Select using querySelector (CSS selector) const firstLink = document.querySelector("a.nav-link"); // Select using querySelectorAll const allLinks = document.querySelectorAll("a.nav-link");

Event Handling

Responding to user interactions with event listeners.

What are Events?

Events are actions or occurrences that happen in the browser, such as:

  • User interactions (clicks, key presses, mouse movements)
  • Browser actions (page loading, form submission)
  • API and data updates (AJAX responses, animations completing)

JavaScript can "listen" for these events and execute code in response.

Adding Event Listeners

The modern way to handle events is by using the addEventListener() method:

// Syntax element.addEventListener("event", function() { // Code to run when the event occurs });
// Example: Button click const button = document.getElementById("myButton"); button.addEventListener("click", function() { alert("Button was clicked!"); });

Common Event Types

Category Event Types Description
Mouse Events click, dblclick, mouseover, mouseout, mousemove Triggered by mouse interactions
Keyboard Events keydown, keyup, keypress Triggered by keyboard interactions
Form Events submit, change, focus, blur Triggered by form interactions
Document/Window Events load, resize, scroll Triggered by document or window actions

The Event Object

When an event occurs, JavaScript automatically passes an event object to the event handler function. This object contains useful information about the event:

button.addEventListener("click", function(event) { // The event object contains information about the event console.log(event.type); // "click" console.log(event.target); // The element that was clicked console.log(event.clientX, event.clientY); // Mouse position });

Event Bubbling and Delegation

Events in the DOM "bubble up" from the target element to its ancestors. This allows for event delegation, where you can attach a single event listener to a parent element that will fire for all matching children, even those added later:

// Instead of attaching click events to each button: document.querySelector(".button-container").addEventListener("click", function(event) { // Check if the clicked element is a button if (event.target.tagName === "BUTTON") { console.log("Button clicked:", event.target.textContent); } });
